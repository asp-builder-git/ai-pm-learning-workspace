## Hard Constraints (non-negotiable)

| Constraint | Source | Impact on Solutions |
|---|---|---|
| **LB is a low-resource language for LLMs** | LLM training data distribution | Generated sentences require verification before publishing. v0 must have a human (Chirag or a native speaker) review output. v1/v2 should include a verification step. |
| **Social platform API access requires separate auth** | Instagram/TikTok/X developer policies | Automated posting requires app registration, review, and approval. For v0: manual posting is fine. For v1+: factor in API auth time (days to weeks for approval). |
| **LOD.lu API rate limits** | LOD.lu service terms | Bulk vocabulary lookups may be rate-limited. The existing lod-api.sh handles single lookups — a batch mode may be needed for v1. |
| **Content copyright — GIFs and images** | Licensing law | Trending GIFs may have license restrictions. GIPHY/Tenor provide license-filtered APIs. For v0 manual: stick to public-domain or clearly licensed sources. |

## Soft Constraints (negotiable with effort)

| Constraint | Source | Effort to Remove | Worth It? |
|---|---|---|---|
| **Canva overlay is manual** | Current workflow | High — building custom overlay tooling | Not for v0. Revisit at v1 if manual Canva becomes a bottleneck. |
| **No analytics pipeline** | Current setup | Medium — integrate social platform analytics APIs or build simple tracking | Yes for v1. For v0, platform-native analytics (Instagram insights, likes, shares) are sufficient. |
| **Single distribution channel** | v0 constraint | Low-Medium — adding more channels | Yes for v2, but premature for v0/v1. Validate on one channel first. |
| **Audio/music not yet integrated** | Not a priority for v0 | Medium — TTS API + music sourcing | Worth testing in v1 if static images show engagement. Vocal pronunciation adds significant value. |

## Integration Requirements

- **Must integrate with**: LOD.lu API (vocabulary source), LLM (sentence generation), media search API (GIPHY/Tenor for v1+), distribution channel (Instagram for v0, programmatic for v1+)
- **Nice to integrate with**: luxembourgish-vocab/ repo (seed content), ElevenLabs TTS (audio), Canva API (overlay automation)
- **Must NOT break**: None — this is a new project with no existing system to maintain

## Data Constraints

- **Data sources available**:
  - LOD.lu API — structured vocabulary data (word, IPA, translation, examples, audio URL)
  - luxembourgish-vocab/VOCAB.md — personal vocabulary list and session history
  - LLM training data — general LB knowledge (limited but usable for simple sentences)
- **Data gaps**:
  - No "trending" data for Luxembourgish content — what's culturally relevant right now?
  - No learner engagement data — what words do learners actually struggle with?
  - No user feedback/retention data until v0 launches
- **Data quality**:
  - LOD.lu: High — professionally maintained dictionary
  - LLM LB sentences: Unknown — must be validated
  - Trending media: GIPHY/Tenor search quality depends on query construction
