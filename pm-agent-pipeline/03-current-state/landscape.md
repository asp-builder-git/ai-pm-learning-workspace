## Architecture Decisions

### [Decision 1: LOD.lu as primary vocabulary source]
- **Context**: Need a reliable, structured source for Luxembourgish vocabulary with IPA, translations, examples, conjugation, and audio.
- **Decision**: Use LOD.lu API as the primary vocabulary backend. CLI wrapper (`lod-api.sh`) already exists in the workspace.
- **Consequences**: Vocabulary selection is constrained to words in LOD's database (which is comprehensive but not exhaustive). API provides structured data (IPA, audio URLs, example sentences) that can feed the content pipeline directly.
- **Still valid?**: Yes — LOD.lu is the best available source for structured LB vocabulary data.

### [Decision 2: Manual content creation for v0]
- **Context**: v0's goal is to test engagement signal, not automation capability.
- **Decision**: v0 uses existing manual tools (LLM via chat, Canva for overlays, direct social media posting).
- **Consequences**: Fast to start (1-2 days). Limits volume to 3-5 pieces for the test. If signal validates, automation (v1) takes over.
- **Still valid?**: Yes — this is the core of the revised recommendation.

### [Decision 3: Instagram as primary v0 distribution channel]
- **Context**: Need a single channel to test organic engagement for LB vocabulary content.
- **Recommendation**: Instagram Reels or static/image posts for v0. Visual + text format maps naturally to the "GIF/image with overlay" concept. Strong organic discovery via hashtags (#Lëtzebuergesch, #Luxembourgish).
- **Alternatives considered**: TikTok (better for video, less for static/image), X (text-forward, worse for visual content), WhatsApp bot (requires build — too heavy for v0).
- **Still valid?**: Decision pending — Chirag may prefer a different channel.

## Gaps

Things that don't exist and would need to be built:

1. **Media sourcing pipeline** — Finding and licensing trending/memorable GIFs and images. GIPHY API exists but trending detection per-language is tricky. For v0: manual search. For v1: GIPHY/Tenor API + keyword mapping.
2. **Text overlay engine** — Combining image + text overlay programmatically. No existing tool in the workspace. For v0: Canva manual. For v1: Python Pillow, or Canva API, or a cloud image manipulation service.
3. **Audio integration** — Attaching music/voiceover to content. For v0: none (static images). For v1: ElevenLabs or other TTS API for pronunciation audio. Music sourcing (royalty-free libraries) is an open question.
4. **Distribution automation** — Auto-publishing to social channels. Each platform has its own API, auth flow, and content policies. For v0: manual posting. For v1: consider a single-channel integration (e.g., Instagram API or a simple website/feed).
5. **Content scheduling / queue** — Need a way to queue and schedule content for regular publishing cadence. For v0: manual. For v1: a simple queue + cron job.
6. **Feedback instrumentation** — No analytics setup exists. For v0: manual (check post views/likes). For v1: consider per-content tracking (UTM params, bitly, or platform-native analytics).
