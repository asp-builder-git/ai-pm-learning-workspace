## Option A: Lean v0 → v1 (Recommended)

**One-liner**: Ship 3-5 manual content pieces in 2 days to validate engagement, then build the minimal automation that produced them.

**External tool candidate:** [MoneyPrinterTurbo](./money-printer-turbo.md) could replace the custom Python build in v1 with a ready-made solution — given a word, it outputs a full MP4 reel with stock footage, TTS, subtitles, and music.

### How it works

**v0 (Days 1-2) — Manual validation:**
1. Pick 5 high-frequency Luxembourgish words from LOD.lu or VOCAB.md
2. Use LLM (DeepSeek via current environment) to generate simple sentences for each — English translation below
3. Search GIPHY/Tenor for a trending GIF per word — pick something memorable
4. Use Canva (free tier) to compose: GIF background + word in Luxembourgish + English translation + IPA from LOD
5. Compile into 5 Instagram posts (static image format, 4:5 portrait)
6. Post to Instagram with hashtags (e.g., #Lëtzebuergesch, #Luxembourgish, #LearnLuxembourgish)
7. Wait 48h — measure: views, likes, saves, shares

**v1 (if v0 signal positive, ~Days 5-14) — Minimal automation:**
1. Write a Python script that:
   - Queries LOD.lu API for word data via lod-api.sh
   - Generates simple sentence via LLM call
   - Searches GIPHY API for relevant GIF
   - Composes image overlay using Python Pillow
   - Outputs ready-to-post content to a local folder
2. Output: 5-10 pieces in one command run (~2-5 min)
3. Distribution: manual Instagram post from generated assets (automated API posting is v2)

### Builds on
- LOD.lu API + existing lod-api.sh CLI wrapper
- LLM access via OpenClaw runtime (DeepSeek v4 Flash)
- Canva (v0) → Python Pillow (v1)
- GIPHY/Tenor public API (free tier, no auth for basic search)
- Instagram (manual posting)

### Effort
- **v0**: XS — 1-2 days
- **v1**: S — 1-2 weeks (Python scripting, API integration)
- **Total pilot**: 2-3 weeks if v0 passes

### Strengths
- Validates the core engagement assumption before any real build
- v1 automation is just "code what you did manually" — low risk
- Uses tools already available in the workspace
- Clear go/no-go decision point after 48h

### Weaknesses
- v0 manual quality depends on Chirag's Canva/design skills — won't look as polished as automated output
- Instagram organic reach is algorithm-dependent — a 48h test may not get enough impressions if hashtags are cold
- v1 Python script is single-purpose — may need rewriting for v2

### Open questions
- Is Instagram the right channel, or would TikTok Reels work better for the GIF+audio format?
- Should v0 include an audio element (pronunciation via ElevenLabs TTS) even if manual?
- What's the minimum viable engagement signal? (e.g., 500 views? 50 likes? 5 saves?)

## Option B: Instagram-first, manual, with inline feedback loop

**One-liner**: Skip v1 automation for now — validate engagement AND content formats through manual publishing over 2-3 weeks, collecting learnings to inform a targeted v1 build.

### How it works
1. Days 1-14: Create and publish 2-3 content pieces per week manually (same format as Option A v0)
2. Vary formats: sometimes static image, sometimes video+GIF, sometimes carousel with multiple words
3. Vary hooks: some posts with humorous words, some with high-frequency utility words, some with cultural references
4. Track per-post: impressions, likes, saves, shares, comments
5. Use comments/DMs as qualitative feedback
6. After 10-12 posts, analyze: which formats work best? Which word categories? Which hooks?
7. Only then build automation that produces the winning format

### Builds on
- Same tools as Option A v0 (Canva, LOD.lu, LLM, manual posting)
- Adds: structured notes in 10-feedback/insights.md

### Effort
- **v0 test phase**: S — 2-3 hours per post × 12 posts = 24-36 hrs spread over 2 weeks
- **No automation built in this phase**

### Strengths
- Richer data than Option A's 3-5 post blast — format experimentation
- No script/code risk — pure content validation
- Builds an audience gradually (more natural than a sudden 5-post dump)
- The learnings directly inform what to automate and how

### Weaknesses
- Slower to reach a go/no-go decision (2-3 weeks vs 2 days)
- Manual creation for 10+ posts is tedious — risk of burnout before getting learnings
- No automation scaffolding — if engagement is positive, you start from scratch on v1
- Format variations muddy the signal — too many variables to know what drove engagement

### Open questions
- Do you have the stamina for 2 weeks of manual content creation?
- Are you optimizing for "fastest learning" (Option A) or "richest learning" (Option B)?
- How do you avoid confirmation bias when analyzing which format "worked"?

## Option C: Full automation from day 1, ship fast

**One-liner**: Build the full content generation pipeline as a Python tool in week 1, generate 20+ pieces in week 2, distribute via Instagram + a simple website in week 3.

### How it works
1. Week 1: Build Python CLI tool that covers the full pipeline:
   - Vocabulary selection from LOD.lu or VOCAB.md
   - Sentence generation via LLM
   - GIF search via GIPHY/Tenor
   - Overlay composition via Pillow
   - Output to organized folders
2. Week 2: Run the tool to generate 20-30 content pieces. Batch review for quality. Fix issues and regenerate.
3. Week 3: Manual posting to Instagram (2 posts/day for 2 weeks). Optionally set up a simple static site as a content gallery.

### Builds on
- Same as Option A v1, but built upfront
- Adds: batch review workflow, quality checks

### Effort
- **L**: 2-3 weeks before first content is published

### Strengths
- Once built, content creation is fast and cheap — can iterate on formats quickly
- Full pipeline is ready for v2 enhancements (analytics, multi-channel)
- Avoids manual grunt work

### Weaknesses
- **Violates the core assumption**: invests 2-3 weeks building automation before knowing if anyone wants the content
- Pipeline quality issues (bad sentences, wrong GIFs) only surface after the build is complete
- Harder to pivot based on learnings — rewiring a pipeline vs. changing a manual process
- Recommended sequence (v0→v1→v2) explicitly deprioritizes this approach

### Open questions
- This option conflicts with the Gate 2 decision. Only viable if Chirag decides the engagement risk is acceptable.

## Recommendation

**Go with Option A** — it's the fastest path to an evidence-based go/no-go decision, which is the entire point of a pilot.

The sequence:
1. **Day 1**: Create 5 posts (manual) → post to Instagram
2. **Day 3**: Check engagement. If signal passes threshold → proceed
3. **Days 4-14**: Build v1 automation (Python CLI tool)
4. **Day 15**: Run v1 to generate next batch. Continue publishing and measuring.

If Day 3 signal is negative, you're out 2 days and a few Canva hours. Not bad for a pilot.
