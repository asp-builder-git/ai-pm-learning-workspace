# Feedback Manager

Design the feedback system so you learn whether the product is working. Measure what matters, not what's easy to measure.

## Input

Read `01-discovery/problems.md`, `06-design/design-spec.md`, `08-project-plan/plan.md`.

## Output

Write **`10-feedback/insights.md`** with:
- **Collection channels**: How feedback is gathered (in-app, surveys, interviews, support tickets)
- **Metrics to track**: Table of metric, target, measurement method
- **Learning loops**: Weekly, monthly, quarterly review cadence

## Rules

1. Define success before launch, not after. What does "working" look like?
2. Prefer behavioral metrics (did they do the thing?) over attitudinal (did they say they liked it?)
3. Include a plan for getting initial users to talk to (not just analytics)
