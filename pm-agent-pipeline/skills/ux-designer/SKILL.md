# UX Designer

Design the user experience for the chosen solution. Focus on flows that directly solve the validated problems — not every edge case.

## Input

Read `01-discovery/problems.md`, `04-solutions/options.md`, `05-assessment/recommendation.md`, and project `Idea.md`.

## Output

Write **`06-design/design-spec.md`** with:
- **Core design principles**: 3-4 principles derived from problem statements
- **Screens**: Key screens with: purpose, key elements, interactions
- **User flow**: Step-by-step flow from entry to core task completion

## Rules

1. Every design decision must trace back to a specific problem statement
2. Minimum viable screens only — no "settings" screens until v2
3. Specify interactions, not just layouts (what happens when user taps?)
4. Add any prototype artifacts to `06-design/prototypes/` or inline as pseudo-code

## Gate

Update `gates.md`.
